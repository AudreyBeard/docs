# Teams

W&B Teams を使用して、ML チームがモデルを迅速に構築できるように中央のワークスペースを作成します。

* **すべての実験を追跡** することで、作業の重複を防ぎます。
* **以前にトレーニングしたモデルを保存し再現** します。
* **進捗と結果を共有** して、上司や協力者と共有します。
* **リグレッション検出** し、パフォーマンスが低下したときに即座にアラートを受け取ります。
* **モデルのパフォーマンスをベンチマーク** し、モデルのバージョンを比較します。

![](/images/app_ui/teams_overview.webp)

## コラボレーティブチームを作成する

1. [**サインアップまたはログイン**](https://app.wandb.ai/login?signup=true) して、無料の W&B アカウントを作成します。
2. ナビゲーションバーの **Invite Team** をクリックします。
3. チームを作成し、協力者を招待します。

:::info
**注意**: 組織の管理者のみが新しいチームを作成できます。
:::

## チームプロフィールを作成する

チームのプロフィールページをカスタマイズして、イントロダクションやパブリックまたはチームメンバーに見えるレポートやプロジェクトを表示します。レポート、プロジェクト、外部リンクを紹介します。

* **最も優れた研究を訪問者に紹介** し、最高のパブリックレポートを展示します。
* **最もアクティブなプロジェクトを紹介** し、チームメンバーが簡単に見つけられるようにします。
* **協力者を見つけるために**、会社や研究室のウェブサイトや公開した論文への外部リンクを追加します。

## チームメンバーを削除する

チーム管理者はチーム設定ページを開き、退会するメンバーの名前の横にある削除ボタンをクリックします。ユーザーが退会した後も、そのユーザーがチームにログ記録した runs は残ります。

## チームの役割と権限
同僚をチームに招待するときにチームの役割を選択します。以下のチーム役割オプションがあります：

- **Admin**：チーム管理者は他の管理者やチームメンバーを追加および削除できます。すべてのプロジェクトの編集と完全な削除権限があります。これは run、プロジェクト、Artifacts、および Sweeps の削除を含みますが、これに限定されません。
- **Member**：チームの通常のメンバー。管理者がメールでチームメンバーを招待します。チームメンバーは他のメンバーを招待できません。チームメンバーは自分が作成した runs と sweep runs のみを削除できます。例えば、メンバー A があり、メンバー B が run をチーム B のプロジェクトからメンバー A が所有する別のプロジェクトに移動した場合、メンバー A はメンバー B が移動した run を削除できません。run を作成したメンバーかチーム管理者のみが run を削除できます。
- **Service (Enterprise-only feature)**: サービスワーカー、run 自動化ツールで W&B を使用するために便利な APIキー。チームのサービスアカウントから APIキーを使用する場合、環境変数 **WANDB_USERNAME** を設定して、run を正しいユーザーに帰属させてください。関連する振る舞いについての詳細は以下を参照してください。
- **View-Only (Enterprise-only feature)**: View-Only メンバーは、チーム内の runs、reports、およびワークスペースなどのアセットを表示できます。レポートをフォローしたりコメントしたりできますが、プロジェクトの概要、レポート、および runs を作成、編集、または削除することはできません。View-Only メンバーには APIキーはありません。
- **Custom roles (Enterprise-only feature)**: カスタムロールにより、組織の管理者は上記の **View-Only** または **Member** ロールを継承し、追加の権限を付加して新しいロールを作成できます。その後、チーム管理者はこれらのカスタムロールをチーム内のユーザーに割り当てることができます。詳細については [こちらの記事](https://wandb.ai/wandb_fc/announcements/reports/Introducing-Custom-Roles-for-W-B-Teams--Vmlldzo2MTMxMjQ3) を参照してください。

:::note
W&B は、チームに複数の管理者がいることを推奨しています。これは、主要な管理者が利用できない場合でも管理操作を継続できるようにするためのベストプラクティスです。
:::

:::note
W&B Server (専用クラウド または 自己管理デプロイメント) を使用している場合、**Custom Roles** 機能を使用するためには最新の企業ライセンスが必要です。
:::

### チーム設定
チームの設定により、チームおよびメンバーの設定を管理できます。これらの権限を使用して、W&B 内のチームを効果的に監督および整理することができます。

| 権限                 | View-Only | Team Member | Team Admin | 
| ------------------- | --------- | ----------- | ---------- |
| チームメンバーを追加 |           |             |     X      |
| チームメンバーを削除 |           |             |     X      |
| チーム設定を管理     |           |             |     X      |

### チームサービスアカウントの振る舞い

* トレーニング環境でチームを設定する際、サービスアカウントを使用して、そのチーム内のプライベートまたはパブリックプロジェクトに runs をログに記録することができます。さらに、環境に **WANDB_USERNAME** または **WANDB_USER_EMAIL** 変数が存在し、参照されたユーザーがそのチームの一員である場合、その runs をユーザーに帰属させることができます。
* トレーニング環境でチームを設定せず、サービスアカウントを使用する際、runs はそのサービスアカウントの親チーム内の命名されたプロジェクトにログされます。この場合も、環境に **WANDB_USERNAME** または **WANDB_USER_EMAIL** 変数が存在し、参照されたユーザーがサービスアカウントの親チームの一員である場合、その runs をユーザーに帰属させることができます。
* サービスアカウントは、異なるチームのプライベートプロジェクトに runs をログすることはできませんが、他のチームのパブリックプロジェクトに runs をログすることはできます。

## 高度な設定

### セキュアストレージコネクタ

:::caution
W&B は現在、バケットの移行をサポートしていません。具体的には以下をサポートしていません：
* チームレベルのバケットから別のチームレベルのバケットへの移行
* W&B マルチテナントバケットからチームレベルのバケットへの移行、またはその逆
:::

チームレベルのセキュアストレージコネクタを使用すると、Teams は独自のクラウドストレージバケットを W&B とともに使用できます。これにより、高度に機密性の高いデータや厳格なコンプライアンス要件を持つチームのために、データのアクセス制御とデータ隔離を強化することができます。

:::info
この機能は Google Cloud Storage バケットおよび Amazon S3 バケットのみで利用可能です。企業チームのみがこの機能を使用できます。企業プランの詳細については、[サポートチーム](mailto:support@wandb.com) にお問い合わせください。
:::

クラウドストレージバケットをプロビジョニングするには、[AWS](https://github.com/wandb/terraform-aws-wandb/tree/main/modules/secure_storage_connector) または [GCP](https://github.com/wandb/terraform-google-wandb/tree/main/modules/secure_storage_connector) 用のセキュアストレージコネクタTerraform モジュールを使用してください。

クラウドストレージバケットは、チーム作成時に一回だけチームに構成します。チームを作成するときに **External Storage** を選択して、クラウドストレージバケットを構成します。プロバイダーを選択し、バケット名とストレージ暗号化キーID (該当する場合) を入力し、**Create Team** を選択します。

バケットアクセスに問題があるかバケットの設定が無効である場合、ページ下部にエラーまたは警告が表示されます。

![](/images/hosting/saas_setup_secure_storage.png)

組織の管理者のみがセキュアストレージコネクタを設定する権限を持っています。複数のチームが同じクラウドストレージバケットにアクセスでき、既存のクラウドストレージバケットをドロップダウンから選択します。
